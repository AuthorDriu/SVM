Сначала работа будет идти над разработкой архитектуры
ВМ будет стековой без регистров
Перед каждой операцией необходимо запихнуть значение в стек


виды стековЖ
    стек значений -> хранит в себе значения

Базовые инструкции:

арифметические операции:

    ADD -> забирает из стека два последних значения и записывает ответ в стек
    SUB -> забирает из стека два последних значения и вычитает первое из второго (по порядку снятия)
    MUL -> забирает из стека два последних значения и перемноджает их
    DIV -> забирает из стека два последних значения и второе на первое

логические операции:

    CMP -> снимает два значения из стека и вычитает их, результат записывает в стек (необходим для лучшего понимания), по факту тоже самое что и SUB

операции контроля потока:

    JZ x -> снимает верхнее значение стека и совершает переход на определенное количество байт (пользователь указывает метку) если оно равно нулю
    JN x -> тоже самое, что и JZ, но только если негативное
    CALL flag -> перемещает командный укатель на указанную метку и записывает в стек возврата точку
    RET -> возвращает командный указатель на позициу с верхушки стека возврата

операции ввода/вывода:
    INPI -> запрашивает число
    INPS -> запрашивает строку
    OUTI -> выводит число
    OUTS -> выводит строку
